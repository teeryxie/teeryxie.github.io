Research infrastructure checklist

Good benchmark work is easier to trust when code, data, and evaluation are explicit.

  • Keep paper, code, dataset, and evaluation scripts linked from one place.
  • Separate benchmark data construction from model evaluation.
  • Track exact model versions, prompts, runtime configuration, and failure cases.
  • Make project pages readable before asking visitors to inspect the repository.
